# Кейс №1: Базовая очистка и математическая модернизация данных

## Блок: Финтех
**Заказчик:** Финансовый директор.

### Бизнес-задача (ТЗ):
> *"Очистить транзакции от черных дыр NULL и грязных минусов возвратов. Вывести ID транзакции, ID пользователя и чистую сумму операции в плюсовом значении без потери данных."*

---

## Исходная структура СУБД (Таблица `raw_transactions`):
*   `tx_id` (INT) — Первичный ключ транзакции.
*   `user_id` (INT) — Уникальный номер клиента.
*   `amount` (INT) — Сумма операции (содержит пропуски `NULL` и отрицательные значения возвратов).

---

## Дескриптор авторского ключа (Логика решения):
Заходим в таблицу `raw_transactions`, выводим на экран `tx_id` и `user_id`, далее при помощи `COALESCE` (в Python: `.fillna(0)`) убираем пустые ячейки `NULL`, заменяя их нулями, чтобы сложение и математика сервера не упали в ошибку. 

После, инструментом `ABS` (в Python: `.abs()`) перехватываем отрицательные минусы возвратов, принудительно трансформируем в чистое положительное значение оборота. На выходе выводим очищенную колонку, присваивая ей достойное новое имя `clean_amount`
